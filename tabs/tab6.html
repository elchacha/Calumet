★ Key features

  • Coverage view — fetch Apex class and trigger coverage from Salesforce; rows with coverage > 75% are highlighted in green.
  • Date filter — restrict the coverage search to classes modified after a given date.
  • Source filters — limit displayed classes to items found in the deploy package, retrieve package, SFDX project, or Calumet project.
  • Extract test classes — export the list of test classes covering the selected items (all, or only those reaching >75% coverage).
  • Detail popup — double-click any row to open a per-class coverage breakdown showing which test classes contribute to coverage.
  • Helper search — enter any class or method name (e.g. MyClass.aMethod) to find which test classes exercise it, using a local cache for speed.
  • Test Suite management — create or select a .testSuite-meta.xml file; add classes to it via the context menu on the Helper table.
  • Limits tab — view governor limit consumption per test method execution (SOQL, DML rows, CPU time, callouts, etc.).
  • Results tab — browse all stored test results; filter by outcome (Pass/Fail), keep only the latest run, or trigger a full test execution on the org.
