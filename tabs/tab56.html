★ Key features

  • Load Apex test classes from a local VS Code project folder or directly from the connected Salesforce org
  • Build a TestSuite by transferring classes: click Add / Remove or double-click items in either list
  • Filter the left-hand list in real time with the search box to quickly locate specific classes
  • Save the composed suite to the VS Code project as a .testSuite-meta.xml file with one click
  • Auto-load the suite when exactly one .testSuite-meta.xml exists in the project testSuites folder
  • Deploy the TestSuite to the connected org via Metadata API; vsCode and Org modification dates are tracked in the info bar
  • Run the TestSuite on the org and automatically open the Apex Test Queue in the browser to monitor progress
  • Load the current org version of a TestSuite back into the editor to compare or update
