FlexiPages

Tab 42

Analyse Lightning page assignments and legacy component usage across your Salesforce org. The tab exposes three views: Profile Overrides — which FlexiPage each profile sees for each object action; FlexiPages — all Lightning pages with their Full-Dynamic (FD) score and Classic Layout (CL) dependency count; and Flexi Unused — Lightning pages that are not assigned to any profile or app. Double-clicking any row opens the corresponding Lightning page in Setup. Data is loaded from a Tooling API cache that is shared across tabs.

Key Features

  • FD badge: counts legacy (non-Full-Dynamic) components on a Lightning page — green FD means fully modern, red ≥7 means heavy legacy debt.
  • CL badge: counts Classic Layout references inside the page — a non-zero value indicates remaining dependencies on old page layouts.
  • Profile Overrides table shows the exact FlexiPage override per profile / record type / action combination, letting you spot inconsistencies at a glance.
  • Layout Dependencies side panel shows, for the selected FlexiPage, all Classic layouts it depends on.
  • Flexi Unused tab identifies Lightning pages that exist in the org but are not assigned to any profile or Lightning app — candidates for cleanup.
  • All three tables support live text filters; data is cached to avoid repeated API calls.

Screenshots

Profile Overrides — each row is a profile/action/record-type override. The FD badge shows the legacy component count for that Lightning page. Blue "View" action badges indicate the page is shown on the record view action.
FlexiPages tab — the main table lists every Lightning page with its FD and CL scores. Selecting a row populates the Layout Dependencies panel on the right with the names of all Classic layouts still referenced.
Flexi Unused — Lightning pages that are not assigned to any profile or Lightning app. The Object Related column links the page back to its Salesforce object, making cleanup straightforward.
