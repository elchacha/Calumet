★ Key features

  • Named configurations — create, clone and switch between multiple seeding configurations, each with its own source/target org pair and main path.
  • Dependency traversal — automatically discovers parent and child relations up to a configurable depth and generates a full traversal plan (traversal_plan.json).
  • Per-object record limits — edit SOQL LIMIT constraints inline per object in the Query Recap table; limits are auto-saved and can be persisted to disk.
  • Export Records — executes all queries in the traversal plan and writes results to CSV files; a dry-run Count Records mode estimates volumes first.
  • Check IDs / Check Users — validates parent ID integrity across all exported CSVs, and verifies that source users have matching profiles in the target org before import.
  • Import with Bulk API v2 — imports all CSVs into the target org in dependency order; orphaned relation rows are remapped automatically using the oldId-to-newId table.
  • Clean with date-range filtering — delete previously imported records from the target org, optionally constrained to a Created After / Created Before window.
  • Automation management — disable all flows and triggers on the target org before import, then re-enable them from a local backup after completion.
  • Community Users support — export and import Experience Cloud community users (Account + Contact + User) with username extension renaming for the target org.
  • Debug modes — IDs Export tab inspects every ID-type field per exported object; Debug Import tab shows the full oldId-to-newId mapping table with user-remap injection support.
  • Import dependency chaining — configure one config to pre-load oldNewIds from another config so shared parent records are not duplicated across linked configs.
