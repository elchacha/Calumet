★ Key features

  • Autocomplete user selector — type any part of the name to filter org users instantly
  • Objects tab: shows all object-level CRUD permissions (Create, Read, Edit, Delete, View All, Edit All) for the chosen user across every accessible SObject
  • Sharing (AccessRight) tab: lists every sharing entry granting the user access to each object — OWD baseline, sharing rules, and profile/permission-set overrides
  • Sharing Rule Name column uses color-coded badges: OWD (orange), Edit All (red), View All (amber), named sharing rules (blue)
  • Access Level column shows the effective access: ReadWrite (orange), Read (blue/grey), Edit (grey), ControlledByParent (dark grey)
  • Reason column explains why access was granted — shows the Profile name or Permission Set name responsible
  • Object filter file lets you restrict display to a curated list of objects — click Manage Object Filter to open and edit the file
  • Tab counters in the header reflect the exact number of loaded entries after each search (e.g. Objects (221) / Shraing (478))
  • Columns auto-resize after each search for optimal readability
