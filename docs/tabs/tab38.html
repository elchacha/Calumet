★ Key features

  • Query Visualforce Page last-used history directly from Production (unavailable on sandboxes, including Full Copy)
  • Cache Production usage data so dependency analysis can be run safely against any sandbox
  • Filter the page list by name to quickly locate candidates for deletion
  • Analyse forward dependencies ("find using") and reverse dependencies ("find used by") for any metadata in the delete list
  • Detect additional orphaned metadata that could also be deleted alongside the selected page
  • Move items between the Delete table and the Found table to refine the scope before generating the package
  • Generate a ready-to-deploy destructive change ZIP (destructiveChanges.xml + empty package.xml) saved to the active project folder
  • Double-click any item to open the raw Apex/VFP source in a text editor for inspection
